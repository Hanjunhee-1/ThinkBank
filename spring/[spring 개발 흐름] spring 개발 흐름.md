# Spring 개발 흐름
'이게 100% 맞다..!' 가 아니라 내가 공부를 해보면서 '이렇게 했더니 프로젝트가 얼추 돌아가긴하네' 라고 느꼈던 것을 정리한 것이다.

※ 요구사항 분석, 개발 프로세스 정립 등등과 같은 소프트웨어공학적인 것들은 제외하고 오로지 Spring 을 통해 백엔드 서버를 개발한다고 했을 때의 흐름이다.

1. URI 개발
    - RESTful API 디자인 철학에 맞추어 URI 와 HTTP 메소드를 결정한다. ex) `특정 사용자가 작성한 게시글을 반환해주는 API 가 필요하다.` --> `GET localhost:8080/posts?userId=123`
    - URI 개발은 Controller 개발과도 연결된다.

2. Repository 개발
    - 사용할 DB 종류에 맞추어 인터페이스를 개발하는 것이다.
    - Repository 는 저장소, 보관소라는 뜻을 가진 영단어처럼 Spring 과 DB 의 객체 사이에서 통신을 담당해준다. 
    - Repository 는 JpaRepository 를 상속받는 인터페이스로 작성하여 사용한다. 
    - 이미 JpaRepository 에서 웬만한 쿼리들은 내장되어있기 때문에 Repository 내부에 따로 함수를 작성할 필요는 없지만 만약 내장되어있지 않은 쿼리를 사용해야 한다면 함수를 작성하여 사용해주면 된다.
    - 내가 작성해야 할 쿼리를 JPA 에 내장된 함수를 통해 간단하게 할 수 있기 때문에 편하다. --> 만약 직접 쿼리를 작성해야 하거나 별도의 최적화가 필요하다면 DAO 를 직접 만들어서 사용해도 된다.

3. Service 개발
    - Controller 에서 받은 요청을 실제로 처리하는 비즈니스 로직을 구현한다.
    - Service 에서 Repository 가 하는 것과 같이 DB 에 접근해도 되지 않을까? --> 이럴거면 Controller 에서 전부 다 해라. 그러니까 안된다.

4. DTO 작성
    - DTO 는 Data Transfer Object 의 줄임말로 DB 에서 찾은 데이터를 객체로 반환할 때 반환할 데이터를 필터링 해주는 역할을 하기도 하고 클라이언트 측으로부터 데이터를 입력받을 때 사용하기도 한다.

5. 테스트 코드 작성
    - JUnit 등과 같은 테스트 러너를 활용하여 테스트 코드를 작성한다.

6. 보안 및 추가 모듈 작성 (AOP, 횡단관심사)
    - 로깅, 트랜잭션, 예외 처리, 인증/인가 등등

7. API 문서 작성
    - API 를 개발했다면 해당 API 를 설명하는 문서를 상세적으로 작성한다.

---

8. 프론트엔드와 연동
    - 프론트엔드와 연동하여 제대로 작성하는지 테스트한다.

9. 배포
    - AWS, Nginx, Docker 등을 활용하여 서비스를 배포한다.
